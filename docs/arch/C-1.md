# C-1 — CoAgent Task Runtime

Date: 2026-04-27

## What landed

`electron/native/coagent/task-runtime.ts` (~140 LoC) —
`CoAgentTaskRuntime` joins the C-0 family as `task` and bridges
`TaskStore` mutations (T-0) to the bus. Every `createTask` /
`setTaskStatus` / `setTaskOutput` / `removeTask` call publishes a
typed event other family members subscribe to.

## Outbound events

| TaskStore op | Event | Payload |
|---|---|---|
| `createTask` | `task.created` | `{id,title,status,origin,bindings,createdAt}` |
| `setTaskStatus` | `task.updated` | `{id,status,error}` |
| `setTaskOutput` | `task.updated` | `{id,output}` |
| `removeTask` (when removed) | `task.deleted` | `{id}` |

Read paths (`getTask`/`listTasks`/`listJobs`) never publish.

## Inbound (opt-in)

`{ applyPeerEvents: true }` makes the runtime subscribe to peer
task events and back-fill its local store — the seam Hermon mirror
(C-1.1) plugs into. Own-source events are filtered out (no echo).
Malformed payloads are silently skipped; reconciliation arrives in
C-1.1.

## Tests (13)

- Outbound: created / updated-status / updated-output / deleted; no
  publish on reads (5)
- Inbound: ignores when disabled; applies when enabled; no echo of
  own publish; malformed payload doesn't crash (4)
- Lifecycle: attach idempotent; detach leaves bus; detach without
  attach is a no-op; mutations before attach still mutate the store (4)
