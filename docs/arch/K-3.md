# K-3 ‚Äî Provenance

Date: 2026-04-27

## What landed

`electron/native/knowledge/provenance.ts` (~150 LoC) ‚Äî append-only
`ProvenanceStore` recording `(actionKind, actionId, author, refs[], parent?)`.
Every material agent action carries a `ProvenanceRecord` so the
chat surface can render citation chips and the audit log can replay
the chain.

## API

```typescript
store.record({ actionKind, actionId, author, refs, metadata?, parent? })
store.get(id)
store.list({ actionKind?, actionId?, author?, since?, until?, limit? })
store.latest(actionKind, actionId)         // newest revision per action
store.history(id)                          // walks parent chain
```

`actionKind` is open: `agent.message`, `agent.tool_call`, `file.write`,
`file.edit`, `task.update`, `deliverable.publish`, `memory.put`, ‚Ä¶

Records are immutable; new revisions issue a new record with `parent`
pointing at the prior. `latest(kind, id)` is the index the UI hits.

## Tests (12)

- record + retrieve round-trip (1)
- reject empty actionId / author (1)
- `latest()` returns most recent revision (1)
- parent chain ‚Äî `history()` newest-first (1)
- reject unknown parent (1)
- list filters by actionKind (single + array), actionId, author (3)
- `since` / `until` time bounds (1)
- newest-first sort + limit (2)
- deep-copy isolation (1)
