## Contract (locked)

A `ProviderAdapter.chat()` MUST yield events matching the canonical
sequence:
1. First event is `response.started` with a `responseId`.
2. Last event is `response.completed` (success) OR `response.error`
   (failure) — never both, never neither.
3. `usage.updated` MAY appear once between started and completed.
4. `AbortSignal` honored within ~100ms; aborted runs end with
   `response.error` `code: "provider.cancelled"`.

`ProviderRuntime.chat()` validates every yielded event through
`AIStreamEventSchema` (A-0 Zod). A malformed event short-circuits to a
`response.error` `code: "provider.protocol_violation"`. Provider-shape
leaks above the seam are impossible by construction.

## Wow moment

`runtime.test.ts` proves the whole abstraction with the test provider:
register → list models → chat → assert canonical event sequence → abort
→ assert `response.error{code:"provider.cancelled"}`. Every real provider
reuses this test scaffold by swapping the adapter.

## Handoff

A-2 (agent runtime) consumes `ProviderRuntime` directly and adds the
turn loop, tool registry, memory adapter, and telemetry hooks. A-3
moves provider-specific extras (Anthropic OAuth / billing-attribution,
Hermon-mediated routing) into provider Cells.
