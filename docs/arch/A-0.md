# A-0 — Canonical AI Types (TS + Rust mirror)

Date: 2026-04-25
Phase: A-0 (Shared AI runtime — canonical types)

## Decision

The single source of truth for AI-related shapes lives at:
- `electron/shared/ai/canonical.ts` (TypeScript)
- `hermon/crates/hermon-types/src/ai.rs` (Rust mirror)

Both surfaces match CONVENTIONS § 4 byte-for-byte on the wire. The TS
side ships Zod schemas at `electron/shared/ai/zod.ts` for runtime
validation at IPC + Hermon-client boundaries.

## Types

| TS shape | Rust shape | Notes |
|---|---|---|
| `AIProvider` | `AIProvider` | `kind` is kebab-case enum |
| `AIModel` + `AICapabilityProfile` | `AIModel` + `AICapabilityProfile` | camelCase wire |
| `AIRole` | `AIRole` | snake_case wire |
| `AIContentBlock` (discriminated union) | `AIContentBlock` (`#[serde(tag = "kind", rename_all = "snake_case")]`) | kinds: text / image / audio / file / tool_use / tool_result / reasoning |
| `AIMessage` | `AIMessage` | |
| `AIToolDefinition` / `AIToolInvocation` / `AIToolResult` | mirror | `unknown` ↔ `serde_json::Value` |
| `AIStreamEvent` (discriminated union) | mirror | kinds: response.started / content.delta / content.completed / tool_call.started / tool_call.delta / tool_call.completed / usage.updated / response.completed / response.error |
| `AIUsage` / `AIError` | mirror | |
| `AIRequest` / `AIResponse` / `AISession` | mirror | |

Provider-NEUTRAL by construction — no field names carry provider id
prefixes. Provider-specific shape conversion happens in provider Cells
(A-3); A-0's job is the spine.

## Wire format invariants (locked)

1. Discriminated unions tag on `kind`, snake-case values.
2. Object field names are camelCase (TS source order).
3. `AIDataRef = string | { url: string }` is an untagged union on Rust
   side (`#[serde(untagged)]`).
4. `unknown` payloads (`AIToolUse.input`, `AIToolResult.output`,
   `AIToolDefinition.inputSchema`) round-trip as raw JSON in Rust.
5. Optional fields use `?` in TS and `Option<_>` in Rust with
   `#[serde(skip_serializing_if = "Option::is_none")]` so absent fields
   omit cleanly.

## TS Zod nuance

The Zod schemas drop their explicit `z.ZodType<X>` annotations because
`z.unknown()` infers as `unknown | undefined` in TypeScript while the
canonical types declare `input` / `output` / `inputSchema` as required.
Runtime validation is unaffected; consumers should rely on
`z.infer<typeof Schema>` or import the canonical type alias rather than a
structural-equality check between the two views.

## Tests

- Rust: `hermon-types::ai` ships 17 tests inside `ai.rs` covering serde
  round-trips for every block kind, every stream event, every shape;
  total `cargo test -p hermon-types` = 50/50 passing.
- TypeScript: Vitest is installed; the round-trip test suite (one per
  shape) is filed as the first deliverable of D-1 (which also lands the
  vitest config and `test` script).
- Cross-language byte-equivalence fixture (`__fixtures__/canonical-roundtrip.json`)
  is deferred — it requires both the TS test runner and a Rust
  integration test reading from the wishcode tree, which crosses repo
  boundaries. Filed as `A-0.1-roundtrip-fixture` follow-up.

## Wow moment

When D-1 lands, every IPC call typed via the canonical types
deserializes safely on both sides without a single provider-shaped
field crossing the boundary.

## Handoff

A-1 (provider runtime) consumes the canonical types and builds the
adapter layer for each provider. A-2 (agent runtime) consumes A-1. A-3
moves provider-specific extras (Anthropic OAuth, etc.) into provider
Cells.
