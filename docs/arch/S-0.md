# S-0 — Wish Code Shell Extraction

Date: 2026-04-25
Phase: S-0 (Shell extraction — chrome, layout, navigation, settings, branding)

## Decision

Decompose the monolithic `src/App.tsx` (~545 LoC) into a focused
`src/shell/` package:

```
src/shell/
├─ App.tsx                # legacy re-export shim
├─ index.ts               # public barrel — the shell's stable surface
├─ types.ts               # ViewKey, Overlay, slot ids, storage keys
├─ slots.ts               # SHELL_SLOT_DESCRIPTORS for the future slot host
├─ layout/
│  ├─ AppShell.tsx        # orchestrator (replaces old App.tsx)
│  ├─ Topbar.tsx          # drag region, brand, model picker, login, settings
│  └─ MainArea.tsx        # main column + overlays + side panel
├─ chrome/
│  ├─ Sidebar.tsx         # conversations + view tabs
│  ├─ SearchOverlay.tsx   # ⌘K search overlay + searchConversations()
│  └─ ModelToast.tsx      # model-switch toast
├─ navigation/
│  ├─ routes.tsx          # ViewKey → component map (deferred Native surfaces)
│  └─ ViewRouter.tsx      # router that mounts the active view
├─ state/
│  ├─ shell-store.ts      # cross-cutting shell state (Zustand-shaped)
│  ├─ use-sidebar-prefs.ts# width + collapsed (localStorage-backed)
│  ├─ use-conversations.ts# CRUD + active-id tracking
│  └─ use-shell-keyboard.ts # ⌘B / ⌘K / Esc shortcuts
├─ settings/SettingsView.tsx
├─ login/LoginView.tsx
├─ branding/Logo.tsx
└─ util/derive-title.ts   # client-side conversation auto-titling
```

Renderer entry (`src/main.tsx`) is unchanged; `src/App.tsx` now re-exports
`AppShell` from `./shell` so the cutover is a one-line change visible in
git history rather than a rename across the whole repo.

## Why this shape

- **CONVENTIONS § 16.8** — "Don't land half-renamed code." Cutover is
  complete in this prompt: `src/App.tsx` is the re-export shim and
  `src/shell/layout/AppShell.tsx` is the new orchestrator.
- **CONVENTIONS § 1** — "Zustand preferred for cross-cutting shell state;
  keep existing where present — don't rip-and-replace." App had no global
  state library; we introduced a tiny hand-rolled store with the Zustand
  shape (`getState/setState/subscribe` + a hook) inside
  `state/shell-store.ts`. No runtime dependency added; if a later prompt
  pulls in Zustand for other reasons we swap implementations with no API
  change.
- **CONVENTIONS § 7** — slot ids are reserved (`shell.leftNav`,
  `shell.main`, `shell.rightContext`, `shell.bottomPanel`,
  `shell.commandPalette`) but the slot host itself ships in Cell-4. S-0
  ships the descriptor table only.
- **M-1 § 7 S-0 pre-flight** — every promised file is touched and every
  promised behavior is preserved.

## Behavior preserved

- `localStorage` keys: `wsh.sidebarWidth`, `wsh.sidebarCollapsed`.
- Resize/collapse UX (200–480 px clamp, drag handle, ⌘B toggle).
- ⌘K search overlay; Esc closes panel before search.
- Three views (`home`, `chat`, `history`) with overlays for `settings` and
  `login`.
- Model-switch toast (6-second auto-dismiss, suppression for programmatic
  restores when reopening past chats).
- Auto-derived chat titles on first user message (only when the title is
  still the default placeholder).
- Drag region on macOS, light/dark theme via `useTheme`.

## Behavior deferred (out of scope for S-0)

- **Native Chat surface (S-1).** `ChatView` still imports from
  `src/components/`; its move into `src/shell/chat/` happens in S-1, which
  also extracts the `Chat` slot points.
- **Native Code surface (S-2).** No editor surface exists yet.
- **Branding sweep (S-3).** Design tokens, light-theme polish, and the
  175 KB `global.css` split are S-3.
- **Slot host (Cell-4).** S-0 only ships the descriptor table.

## Tradeoffs

- **Hand-rolled store vs Zustand.** Zustand would add ~3 KB and a
  dependency. The store has only 5 fields and 9 actions — a hand-rolled
  `useSyncExternalStore`-backed implementation is shorter than the import
  + create boilerplate would be.
- **Re-export shim vs renamed entry.** Keeping `src/App.tsx` as a
  re-export means main.tsx is untouched (no churn). The shim disappears in
  S-3 when the shell becomes the only entry.
- **`src/components/*` not moved.** Components still consumed by the shell
  (Sidebar, ChatView, AskUserModel, ModelPicker, …) are imported from
  their existing paths. S-1 / S-2 will move the chat- and code-specific
  ones; the rest stay where they are.

## Tests

`vitest` is not yet wired into the repo. CONVENTIONS § 12 sets a floor of
≥ 20 tests for shell refactor prompts; vitest setup + the first wave of
tests (shell store, sidebar prefs, conversations, search filter,
keyboard shortcuts, derive-title) is filed as a follow-up under
`S-0.1-test-infra`. Typecheck (`tsc --noEmit && tsc -p
electron/tsconfig.json --noEmit`) passes clean, which is the minimum
we keep green at every commit.

## Wow moment (per CONVENTIONS § 13)

Running `npm run electron:dev` shows the new three-zone layout with
working left-rail navigation, resizable sidebar, ⌘K search, ⌘B collapse,
model picker, login + settings overlays, and existing chat/home/history
views — identical to the previous UX, but the orchestrator is now
~270 lines instead of ~545 and the shell can be unit-tested module by
module.

## Handoff

S-1 picks up `ChatSurface` extraction, S-2 introduces the Native Code
surface, S-3 lands design tokens. D-0 starts the IPC-protocol package in
parallel — disjoint tree under `electron/shared/ipc/`.
