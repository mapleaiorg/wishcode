## Decision

`JobOrchestrator` drives a `TaskStore`'s ready jobs through a
`RunnerRegistry`, applying retry-with-attempts, cascade-skip on
failure, and Tel-0 telemetry hooks. Pure logic тАФ runners own the
actual work.

## Operating model

```
tick(taskId)
  тФЬтФА if signal.aborted тЖТ bail
  тФЬтФА load task; if no ready jobs тЖТ maybeFinalizeTask + return
  тФЬтФА promote queued task тЖТ running
  тФЬтФА for each ready window (size = concurrency):
  тФВ     await Promise.all(window.map(runOne))
  тФФтФА maybeFinalizeTask
```

`runOne(job)`:
- look up runner by `job.kind`
- if missing: setStatus running тЖТ failed{code:"orchestrator.no_runner"};
  cascadeSkip dependents; emit `job.run.no_runner`
- else: setStatus running, bumpJobAttempt, emit `job.run.started`
- call `runner.run(job, { signal })`; thrown errors become structured
  failures with `code:"runner.threw"`
- if `state` returned, persist via `setJobState`
- on success: setStatus succeeded; emit `job.run.succeeded`
- on failure: setStatus failed (skip-cascade); emit `job.run.failed`
  with `retried` flag based on `attempts < maxAttempts`

`maybeFinalizeTask` checks every job is terminal then transitions the
Task to `succeeded` (all jobs OK) or `failed` (any failed/skipped).
Illegal-transition errors from the store are swallowed тАФ the store
refused, the orchestrator respects.

## RunnerRegistry

Plain `Map<JobKind, JobRunner>`; `register` rejects duplicates. Cell-3
will supply Cells with a wrapper that auto-`unregister`s on Cell unload.

## TickResult

`{ started, succeeded, failed, skipped, retried }` тАФ telemetry-friendly
counts useful in dashboards and tests.

## drain()

Convenience for tests + synchronous callers: ticks until no more jobs
start (fixed-point), with a 100-iteration safety cap.

## Telemetry events emitted

- `job.run.started` (info)
- `job.run.succeeded` (info)
- `job.run.failed` (warn or error depending on retry status)
- `job.run.no_runner` (error)

All carry `kind`, `jobId`, `taskId` тАФ no payloads. Tel-0 scrub catches
accidental leaks.
