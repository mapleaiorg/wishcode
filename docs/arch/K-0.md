# K-0 — Knowledge Registry

Date: 2026-04-26
Phase: K-0 (Knowledge — registry + provenance shape)

## Decision

Every retrievable artifact registers as a `KnowledgeSource` in the
`KnowledgeStore`. K-1 (indexer) walks each source to produce BM25 +
vector chunks; K-2 (retrieval) reads those chunks; K-3 (provenance)
attaches `ProvenanceRef`s to material agent actions for citation.

K-0 ships only the registry and the provenance reference shape. No
indexer, no retrieval — those build on this contract.

## Source shape

| Field | Notes |
|---|---|
| `id` | server-minted (`src_…`) |
| `slug` | stable human id, lowercase alnum + `_./-` |
| `kind` | `fs.directory` / `fs.file` / `git.ref` / `connector` / `inline` / `url` |
| `uri` | per-kind backend location |
| `scope` | `{ workspaceId?, teamId?, personal? }` mirroring `MemoryBindings` |
| `capabilities` | `{ bm25, vector, hashable }` set by the indexer |
| `status` | `unindexed | indexing | indexed | errored | disabled` |
| `title` | shown in citation chips |
| `description?` | one-line, ≤ 280 chars |
| `contentHash?` | indexer-set; drives change detection |
| `chunkCount?` | indexer-set; for the settings UI |
| `error?` | set on `errored` |

## Provenance ref shape

```typescript
interface ProvenanceRef {
  sourceId: string
  chunkId?: string                            // empty = whole source
  span?: { from: number; to: number }         // byte / line slice
  score?: number                              // K-2 retrieval rank
  hints?: Record<string, unknown>             // snippet, headings, …
}
```

K-3 will attach arrays of these to every material action; the chat
surface (S-1) renders them as citation chips beneath the assistant
message.

## Wow moment

A handful of `registerSource` calls + a `listSources({ scope: { workspaceId } })`
returns precisely the workspace-scoped sources, ready for K-1 to
schedule indexing without revealing personal/team sources to the wrong
workspace.

## Handoff

K-1 reads this ADR + walks `listSources({ status: ['unindexed'] })`,
calls `markIndexing()`, hands chunks to the indexer crate, and on
completion calls `markIndexed({ chunkCount, contentHash })` or
`markErrored(...)`. K-2 reads sources by scope to filter retrieval.
