# D-1 — Preload Bridge Migration

Date: 2026-04-25
Phase: D-1 (Domain — preload uses D-0 schemas; vitest infra; first test wave)

## What landed

- `electron/preload.ts` updated:
  - Lazy `wish:proto:version` handshake on first `invoke()`. Renderer
    surfaces a `WishError("protocol_violation")` on mismatched
    `IPC_PROTOCOL_VERSION`; tolerates a missing handshake on legacy main
    until D-2 lands.
  - Every response goes through `getChannelEntry(channel).response.safeParse(...)`.
    Validation failures throw `WishError("protocol_violation")` with the
    Zod flattened error attached as `cause`.
  - Legacy `{ ok, value }` envelope still accepted at the wire — D-2
    flips main to the canonical `{ ok, data }` shape.
- `electron/shared/ipc/registry.ts`: single `IPC_REGISTRY` map keying every
  channel id to `{ request, response }`. Exposes `IpcChannelId`,
  `registryChannels()`, `getChannelEntry()`.
- `vitest.config.ts` at repo root + `npm test` script.
- First test wave under `electron/shared/ipc/__tests__/`:
  - `registry.test.ts` (9 tests) — every M-0 channel + protocol-version
    handshake + `wish:<domain>:<action>` shape + uniqueness + lookup.
  - `error.test.ts` (8 tests) — `WishError` ↔ `WishErrorShape` round-trip,
    `Result<T>` discriminated union via `resultSchema`.

Total D-1 tests: **17 passing**. Combined with A-1's runtime tests (7),
the wishcode test suite is at **24/24 passing** on a fresh `npm test`.

## Wow moment

`await window.wish.app.version()` flows through the bridge with runtime
validation. A deliberately bad payload (e.g. server returns an extra
field) raises `WishError("protocol_violation")` with a structured
flattened-error `cause` — no more silent drift.

## Pending (deferred to D-2)

- Main-process handler migration to consume the same registry — handlers
  currently live in `electron/main.ts` and `electron/native/*` and use the
  legacy `{ ok, value }` envelope.
- Strict event-channel validation on subscriptions.
- Per-channel exhaustive request/response fixtures (pair with each
  domain's Cell tests in C-* / Cell-*).

## Handoff

D-2 reads this ADR + `electron/main.ts` channel wiring and migrates each
handler to validate its input through the registry's request schema +
return the canonical envelope. After D-2, the legacy `{ ok, value }`
fallback in preload can be deleted.
