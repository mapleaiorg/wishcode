# K-1 — Knowledge Indexer (BM25)

Date: 2026-04-26

## What landed

`electron/native/knowledge/indexer.ts` (~270 LoC) — `BM25Indexer`
walks a `KnowledgeSource`'s content into chunks (default 500 chars
with 80 char overlap, prefer paragraph → sentence → space breaks),
builds an inverted index, and answers BM25-ranked queries scoped by
source.

## Pipeline

```
indexer.index(source, content)
  ├─ removeSource()                     idempotent re-index
  ├─ store.markIndexing(source.id)
  ├─ chunkText(content, ...) → pieces
  ├─ for each piece:
  │     tokens = tokenize(piece)
  │     update postings + total dl
  └─ store.markIndexed(source.id, { chunkCount, contentHash })

indexer.search(query, opts)
  ├─ tokenize(query)
  ├─ for each query token:
  │     get posting list (chunk ids)
  │     compute BM25 contribution per chunk
  └─ rank by score; attach ProvenanceRef
```

## BM25 parameters

`k1 = 1.5`, `b = 0.75` (defaults; `IndexerOptions` lets the host
tune). IDF uses the smoothed `log(1 + (N - df + 0.5) / (df + 0.5))`
form.

## Output → `SearchHit`

```typescript
{
  chunk: { id, sourceId, uri, text, tokens, tf },
  score: number,
  ref: ProvenanceRef {
    sourceId, chunkId, score,
    hints: { uri, snippet }   // first 200 chars
  }
}
```

The `ref` is exactly the shape K-3 attaches via the Cell-3 SDK's
`knowledge.cite(ref)`.

## Tests (18)

- `tokenize`: lowercases / strips punctuation / drops stop words /
  keeps alphanumerics (2)
- `chunkText`: empty, short, long-with-paragraph-breaks (3)
- Indexer: single source index + chunk count, store status update,
  empty-query no-hits, finds query terms, descending score order,
  `sourceIds` filter, `minScore` floor, `limit`, `ProvenanceRef`
  shape, reindex replaces old chunks, `removeSource` drops postings,
  vocabulary size (12)
- `markIndexed` carries `chunkCount` + `contentHash` back to the store (1)

## Wow moment

`indexer.search('argon2id verify')` against a Hermon-auth doc returns
the password-hashing chunks with a `ProvenanceRef` ready for the
Cell SDK `knowledge.cite()` — the citation chain from indexer to UI
is one method call long.
